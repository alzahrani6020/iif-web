# تكامل IIF + المنصة الحكومية + محرك البحث

## لماذا محرك + فهرس محلي؟ («البيانات الكاملة»)

المنصة مصممة على **نموذجين مكمّلين** — لا يُستبدل أحدهما بالآخر:

| الطبقة | الدور |
|--------|--------|
| **الفهرس المحلي** (`government-data.json` → `government-data.js`) | هيكل مرجعي ثابت: جهات، دول، روابط وعناوين يمكن صيانتها ومراجعتها يدوياً أو عبر سكربتات التوسعة (مثل `expand-government-data-to-un195.mjs`). |
| **محرك SearXNG** (`engines/searxng/`) | **إكمال** الاستعلام: مصادر ويب حديثة، تنوّع المحركات، ونتائج لا يمكن تخزينها كلها في JSON. |

- **لا يوجد ملف JSON يحوي «كل بيانات العالم»** — ذلك غير عملي وغير محدّث؛ لذلك **المحرك هو طبقة الاستعلام المفتوح** بجانب الفهرس.
- على **`localhost` / `127.0.0.1`** يُفعّل **تضمين ويب** افتراضياً (يمكن تعطيله من الواجهة أو عبر `?web=0` في الرابط).

## الوضع الحالي

| الموضوع | الحالة |
|---------|--------|
| **محرك SearXNG** | مشروع تحت **`engines/searxng/`** — Docker؛ الإعدادات في **`config/settings.yml`**. |
| **`iif-config.js`** | **يُحمَّل** من `SIMPLE-GOVERNMENT-PLATFORM.html` — يفعّل **`searxngEnabled`** تلقائياً على **`localhost` / `127.0.0.1`** فقط؛ يُعطّل على أي نطاق آخر (مثل Netlify). يمكن التجاوز بـ `window.__IIF_CONFIG__`. |
| **المنصة** | بحث محلي دائماً؛ **بحث ويب** عبر المحرك عند التوفر (افتراضياً على الجهاز المحلي) + **`npm start`** (بروكسي `/api/searx`). في الرابط: `?web=1` يفرض التضمين، `?web=0` يعطّله. |
| **خادم التطوير** | **`npm start`** — بروكسي **`/api/searx`** → `127.0.0.1:18080`. |

## تشغيل المحرك

**ويندوز (سريع):** انقر نقراً مزدوجاً **`engines/searxng/start-searxng.bat`** (يتطلب Docker Desktop).

**أو من الطرفية:**

```bash
cd engines/searxng
docker compose up -d
```

→ [http://127.0.0.1:18080](http://127.0.0.1:18080)

## الإنتاج (Netlify وما شابه)

- لا يوجد بروكسي SearXNG افتراضياً على الاستضافة الثابتة — **بحث الويب عبر المحرك يبقى معطّلاً** ما لم تُضف **دالة Netlify** أو **خادماً** يوجّه إلى مثيل SearXNG.

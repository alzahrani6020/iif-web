# قائمة تحقق يدوية — واجهة الصندوق ولوحة التحكم

استخدم **نفس العنوان دائماً:** `http://127.0.0.1:3333` (تجنّب `localhost` إن لاحظت اختلاف الجلسة).

شغّل الخادم من جذر المستودع: `npm start`

---

## بعد كل تعديل على الدخول أو اللوحة

| # | الخطوة | المتوقع | نعم / لا |
|---|--------|---------|----------|
| 1 | افتح `/panel` | تحويل أو تحميل الصفحة مع بوابة الإدارة؛ تظهر واجهة الدخول إن لم تكن مسجّلاً | |
| 2 | سجّل الدخول (بريد مالك/مسؤول كما في `IIF_CONFIG`) | تختفي أزرار «تسجيل الدخول / إنشاء حساب» من الهيدر | |
| 3 | بعد الدخول | تظهر بطاقة المستخدم في الهيدر (اسم أو بريد) | |
| 4 | اللوحة | تُفتح تلقائياً **أو** يوجد زر/مسار يفتح `#dashboard-overlay` | |
| 5 | حدّث الصفحة (F5) | تبقى حالة «مسجّل دخول» كما في 2–3 | |
| 6 | افتح الجذر `/` بدون `/panel` | إن كنت مسجّلاً: الهيدر يظل يعرض المستخدم (لا يطلب الدخول من جديد) | |
| 7 | (اختياري محلي) `/admin-direct` | يضبط جلسة التطوير ويفتح اللوحة — **للتطوير فقط** | |

## أوامر آلية مرتبطة

- فحص HTML سريع: `npm run qa:iif-fund`
- فحص مع تشغيل الخادم: `npm run smoke:with-server`
- فتح اللوحة في المتصفح (E2E): `npm run e2e:dashboard`  
  (يتطلب: `npm install` ثم `npx playwright install chromium` مرة واحدة)

# اربط Netlify بمستودع GitHub — خطوات من المتصفح

> لا يمكن ربط الحساب من Cursor؛ نفّذ الخطوات التالية في **المتصفح** وأنت مسجّل الدخول إلى Netlify.

## المستودع المطلوب

- **GitHub:** `alzahrani6020/iif-web`
- **الفرع:** `main`

---

## إذا رأيت في Netlify «Build from drop deployment»

هذا يعني أن الموقع نُشر **بسحب ملفات** إلى Netlify — **بدون GitHub**.  
للربط التلقائي مع مستودع `iif-web` تحتاج إما **ربط المستودع** لنفس الموقع (من **Build & deploy → Link repository**) أو إنشاء موقع جديد بـ **Import from Git** كما بالأسفل.

---

## إن كان عندك موقع `fluffy-meerkat-eff966` (أو أي اسم)

1. افتح: [لوحة المواقع](https://app.netlify.com/teams/dr-talal/projects) واضغط على **اسم الموقع**.
2. **Site configuration** → **Build & deploy** (أو **Project configuration** حسب واجهة Netlify).
3. قسم **Continuous deployment**:
   - إن رأيت **Link repository** أو **Configure** أو **Connect to Git** → اضغطه.
   - اختر **GitHub** → سجّل الدخول / امنح Netlify صلاحية الوصول للمستودعات.
   - اختر **`alzahrani6020/iif-web`** ثم الفرع **`main`**.
4. **إعدادات البناء** (أو بعد الربط مباشرة):
   - **Base directory:** اتركه **فارغاً**.
   - **Build command:**  
     `npm ci --no-audit --fund=false && npm run build`  
     (أو اتركه فارغاً إن قرأ Netlify من **`netlify.toml`** تلقائياً.)
   - **Publish directory:** `.`
5. **Save** ثم **Deploy site** أو **Trigger deploy** → **Clear cache and deploy site** (مفضل لأول مرة بعد الربط).

---

## إن لم تجد «ربط مستودع» بسهولة

1. من [app.netlify.com](https://app.netlify.com): **Add new site** → **Import an existing project**.
2. **GitHub** → اختر **`iif-web`** → الفرع **`main`**.
3. انسخ الإعدادات من [NETLIFY-UI-SETTINGS.md](./NETLIFY-UI-SETTINGS.md) إن طلبت الحقول يدوياً.
4. بعد النشر، يمكنك حذف الموقع التجريبي القديم أو تركه — المهم أن **موقعاً واحداً على الأقل** مربوط بـ `iif-web`.

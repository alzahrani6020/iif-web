# خطة تطوير محرك البحث (SearXNG)

> المحرك **منفصل** عن صفحات المنصة إلى أن نكمل الاختبار. المسار: `engines/searxng/`

---

## المرحلة 1 — التشغيل والتحقق (أسبوع أول)

| # | المهمة | التفاصيل |
|---|--------|----------|
| 1 | تثبيت **Docker Desktop** على ويندوز | يجب أن يعمل (أيقونة الحيتان) قبل `docker compose`. |
| 2 | تشغيل المحرك | من هذا المجلد: `docker compose up -d` |
| 3 | فتح الواجهة | [http://127.0.0.1:8080](http://127.0.0.1:8080) — جرّب بحثاً بسيطاً. |
| 4 | مراقبة السجلات | `docker compose logs -f` — تأكد من عدم أخطاء فادحة. |

**معيار النجاح:** البحث يعيد نتائج من محركات مفعّلة في SearXNG الافتراضية.

---

## المرحلة 2 — الضبط (محلياً) ✅

| # | المهمة | التفاصيل |
|---|--------|----------|
| 1 | **`config/settings.yml`** | مفعّل مع `docker-compose` (مجلدا `config` و`data`). راجع [README.md](./README.md). |
| 2 | تقليل المحركات / لغات إضافية | عدّل `settings.yml` أو واجهة SearXNG — [التوثيق الرسمي](https://docs.searxng.org/admin/settings/settings.html). |
| 3 | اختبار الاستقرار | ترك المحرك يعمل ساعات وتكرار البحث. |

---

## المرحلة 3 — جاهزية للربط مع المنصة ✅ (جزء محلي)

| # | المهمة | التفاصيل |
|---|--------|----------|
| 1 | **بروكسي التطوير** | `npm start` يوجّه `/api/searx` → `127.0.0.1:8080` (انظر `scripts/dev-server.js`). |
| 2 | **المنصة** | على `localhost`/`127.0.0.1` يُفعّل بحث الويب تلقائياً عند فتح الصفحة من خادم التطوير. |
| 3 | **الإنتاج** | Netlify: بحث ويب يحتاج بروكسي/دالة لاحقاً — راجع [README-IIF-INTEGRATION.md](../../financial-consulting/government-search/README-IIF-INTEGRATION.md). |

---

## أوامر سريعة (من `engines/searxng/`)

```bash
docker compose up -d      # تشغيل
docker compose ps           # حالة الحاوية
docker compose logs -f      # السجلات
docker compose down         # إيقاف
```

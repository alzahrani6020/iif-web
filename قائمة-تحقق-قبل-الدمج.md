## 2) على الجهاز (قبل `git push`)

- [ ] `npm run verify` — يمر بدون خطأ.
- [ ] عند تغييرات واسعة أو قبل دمج مهم: `npm run verify:full`.
- [ ] تغييرات **حرجة** على واجهة الصندوق: راجع [QA-PRE-RELEASE.md](./financial-consulting/iif-fund-demo/QA-PRE-RELEASE.md).
- [ ] إن اعتمدت على SearXNG محلياً (Docker): `npm run verify:stack`.
- [ ] **PowerShell:** فضّل `;` بين الأوامر بدل `&&` إن ظهرت أخطاء.

تفاصيل أوامر إضافية: [AGENTS.md](./AGENTS.md) (قسم «صيانة دورية»).

## 7) تلافي الخلل والاستعادة (كود، جهاز، منصات، برامج)

### وقاية — قبل التعديل أو «التنظيف»

| الموقف | ماذا تفعل |
|--------|-----------|
| تجربة قد تفسد شيئاً | أنشئ **فرعاً** (`git checkout -b …`) أو **`git commit`** لحالة مستقرة أولاً. |
| بيانات الجهات | عدّل **`government-data.json`** فقط ثم **`npm run build`** — لا تلمس **`government-data.js`** يدوياً. |
| حذف أو تنظيف | لا تحذف **`node_modules`** إلا إن كنت ستشغّل **`npm ci`** (أو `npm install`) بعدها مباشرة. |
| محرر / نسخ | **احفظ الملفات** قبل الاعتماد على Git أو المقارنة؛ ما في الذاكرة قد لا يطابق القرص. |

### استعادة — عند فشل التراجع أو توقف جزء من المشروع

1. من جذر المشروع: **`git status`** — هل هناك تعديلات غير مرغوبة؟
2. لإلغاء كل التغييرات غير المُكمَلة (حذر: يضيع ما لم يُحفَظ في commit): **`git restore .`**
3. أعد بناء المُولَّد والتحقق: **`npm ci`** ثم **`npm run build`** ثم **`npm run verify`** (أو **`npm run verify:full`**).
4. إن بقي خلل: تأكد أنك على **الفرع الصحيح** وأن **المجلد المفتوح** هو جذر المشروع وليس نسخة قديمة على قرص آخر.

### الجهاز وويندوز والطرفية

- **PowerShell:** إن فشل ربط الأوامر، استخدم **`;`** بدل **`&&`** أو نفّذ أمراً واحداً في كل سطر.
- **المنفذ 3333 مشغول:** عيّن منفذاً آخر مثلاً `set PORT=3334` ثم `npm start`، أو أغلق العملية التي تستخدم المنفذ.
- **نسخ المشروع** بين C: وD: أو أجهزة: بعد الفتح شغّل **`npm ci`** و **`npm run verify`** مرة واحدة على تلك النسخة.

### المنصات (GitHub / Netlify / Vercel)

- **أسرار:** لا تلصق توكنات في الكود؛ راجع [SECURITY.md](./SECURITY.md).
- **CI:** تغييرات **`.md` فقط** قد **لا تشغّل** workflow الـ CI (راجع [NETLIFY-FREE-TIER.md](./NETLIFY-FREE-TIER.md) و`.github/workflows/ci.yml`) — للثقة الكاملة شغّل **`npm run verify`** محلياً.
- **ازدواجية نشر Vercel:** [VERCEL-DEPLOY.md](./VERCEL-DEPLOY.md) — اعتمد مسار نشر واحداً إن أردت تلافي الالتباس.

### Docker و SearXNG و Tor

- إيقاف **Docker** أو الحاويات يعطل البحث عبر **`/api/searx`** محلياً دون أن يفسد الملفات — أعد تشغيل **`engines/searxng`** حسب [AGENTS.md](./AGENTS.md).
- للتحقق بعد التشغيل: **`npm run verify:stack`**.

# قائمة تحقق قبل الدمج — جهاز، منصات، مساعد

هدفها تقليل **الأخطاء** و**الازدواجية** والحفاظ على **انسجام** العمل بينك وبين الجهاز (محلياً) والمنصات (GitHub / Netlify / Vercel) وبين المساعد في Cursor.

---

## 1) مصدر حقيقة واحد

| الموضوع | المرجع |
|---------|--------|
| روابط الإنتاج | [CANONICAL-URLS.md](./CANONICAL-URLS.md) |
| بيانات الجهات | `government-data.json` ثم `npm run build` — لا تعدّل `government-data.js` يدوياً |
| نشر Vercel / ازدواجية محتملة | [VERCEL-DEPLOY.md](./VERCEL-DEPLOY.md) |
| أسرار وتوكنات | [SECURITY.md](./SECURITY.md) |

---

## 2) على الجهاز (قبل `git push`)

- [ ] `npm run verify` — يمر بدون خطأ.
- [ ] عند تغييرات واسعة أو قبل دمج مهم: `npm run verify:full`.
- [ ] تغييرات **حرجة** على واجهة الصندوق: راجع [QA-PRE-RELEASE.md](./financial-consulting/iif-fund-demo/QA-PRE-RELEASE.md).
- [ ] إن اعتمدت على SearXNG محلياً (Docker): `npm run verify:stack`.
- [ ] **PowerShell:** فضّل `;` بين الأوامر بدل `&&` إن ظهرت أخطاء.

تفاصيل أوامر إضافية: [AGENTS.md](./AGENTS.md) (قسم «صيانة دورية»).

---

## 3) GitHub بعد الدفع

- [ ] **Actions → CI** على آخر commit **أخضر** (نجح).

---

## 4) المنصات (نشر)

- [ ] لا تُكرّر نشراً غير مقصود: إن كان Vercel مربوطاً بـ Git **و** يعمل workflow النشر من Actions، راجع قسم «ازدواجية نشر محتملة» في [VERCEL-DEPLOY.md](./VERCEL-DEPLOY.md) واعتمد مساراً واحداً إن رغبت.
- [ ] بعد النشر: تحقق سريع من روابط [CANONICAL-URLS.md](./CANONICAL-URLS.md) إن طبّقت تغييراً يمس الإنتاج.

---

## 5) العمل مع المساعد (Cursor)

- [ ] المجلد المفتوح = جذر المشروع الفعلي (مثلاً نسختك من `iif-fund-demo`).
- [ ] حدّد المهمة بوضوح (توثيق فقط مقابل تغيير سلوك واجهة).
- [ ] إن اقترح المساعد ما يخالف [AGENTS.md](./AGENTS.md) أو مصدر البيانات أعلاه، ارجع للمرجع لا للاقتراح فقط.

---

## 6) «أكمل اللازم»

عند طلب إكمال تقني كامل: [اكمل-اللازم-قائمة.md](./اكمل-اللازم-قائمة.md).

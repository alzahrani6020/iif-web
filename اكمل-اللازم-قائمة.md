# عندما تقول «أكمل اللازم» — ماذا يجب أن يُنجَز؟

هذا الملف **تعريف مشترك** بينك وبين المساعد حتى لا يبقى عمل نصف مكتمل.

---

## ما المقصود بـ«اكتمال» المهمة التقنية؟

| # | البند | ملاحظة |
|---|--------|--------|
| 1 | **البناء يمر** | `npm ci` و `npm run build` أو **`npm run verify`** (يشمل فحص صياغة `dev-server` و`check-urls`) بدون خطأ. |
| 2 | **Git نظيف** | `git status` لا يظهر ملفات معدّلة أو ناقصة بعد الدمج. |
| 3 | **الرفع إلى GitHub** | `git push origin main` إن وُجد `remote` ولم يُرفض الدفع. |
| 4 | **المراجع صحيحة** | أي رابط في `README` أو التوثيق يشير لملف **موجود** في المستودع. |
| 5 | **التوثيق** | إن أُضيفت ميزة: يُذكر في `README` أو `AGENTS.md` إن لزم. |
| 6 | **ما لا يمكن من البيئة** | إن فشل أمر (صلاحيات، شبكة، تعارض merge، أو **فشل تشغيل الطرفية في Cursor**): يُذكر **صراحة** في الرد وليس يُترك صامتاً. |

---

## إذا تعذّر على المساعد تشغيل `npm` في الطرفية

أحياناً تفشل أوامر المساعد بـ *spawn aborted*. **على جهازك** من جذر المشروع:

```bash
npm run verify
npm start
```

ثم في طرفية ثانية:

```bash
npm run health
```

(`health` = `check-urls` + `smoke:html` — يتطلب الخادم على المنفذ `3333`.)

---

## حدود (ما لا يُعد «ناقصاً» من جهة المساعد)

- **حساباتك** (Netlify login، GitHub token): لا يستطيع المساعد استبدالك فيها.
- **إنشاء ملفات محجوزة** (مثل `.cursorignore` أحياناً): يُوضَح البديل (نسخ من `cursorignore.example`).

---

## للمساعد (Cursor / AI)

عند طلب «أكمل اللازم» أو «اعمل اللازم»:

1. نفّذ أو تحقق من البنود أعلاه بالترتيب المنطقي.
2. إن توقف شيء: اذكر **السبب** و**ما بقي على المستخدم**.
3. لا تُنهِ الرد بـ«تم» إن **لم** يُدفَع إلى GitHub أو فشل البناء دون توضيح.

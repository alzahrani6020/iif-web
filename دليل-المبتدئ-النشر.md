# دليل المبتدئ: من جهازك إلى الإنترنت

> اقرأ من الأعلى للأسفل — لا حاجة لفهم كل شيء دفعة واحدة.

---

## ماذا نريد أن نحقق؟

| كلمة | معناها ببساطة |
|------|----------------|
| **المشروع** | مجلد الملفات على جهازك (`iif-fund-demo`) — موقعك قبل أن يظهر على الإنترنت. |
| **GitHub** | موقع يحفظ **نسخة من الملفات** في السحابة (مثل «صندوق أرشفة» + تاريخ التعديلات). غالباً ما نربطه بباقي الأدوات. |
| **الرفع / Push** | نقل الملفات **من جهازك إلى GitHub** (مرة أو عند كل تحديث). |
| **Netlify** | خدمة تأخذ الكود من GitHub وتجعله **موقعاً يفتحه أي شخص بالرابط** (مثل `something.netlify.app`). |
| **المستودع (Repository)** | «مجلد المشروع» داخل GitHub؛ له عنوان (رابط) على الويب. |

**الترتيب المنطقي:**  
جهازك → **GitHub** (حفظ الكود) → **Netlify** (تحويله لموقع مفتوح للجميع).

---

## الطريقة الأسهل للمبتدئين (بدون أوامر معقدة)

### أ) تثبيت أدوات بسيطة

1. **Git** — من [git-scm.com/download/win](https://git-scm.com/download/win)  
   (يسمح لجهازك «التحدث» مع GitHub.)

2. **GitHub Desktop** — من [desktop.github.com](https://desktop.github.com)  
   (واجهة أزرار بدل كتابة أوامر في الطرفية — **مُنصى به للمبتدئين**.)

### ب) إنشاء حساب على GitHub

- سجّل في [github.com](https://github.com) إن لم يكن لديك حساب.

### ج) رفع المشروع لأول مرة (بـ GitHub Desktop)

1. افتح **GitHub Desktop**.
2. **File** → **Add Local Repository** → اختر المجلد:  
   `C:\Users\vip\iif-fund-demo`
3. إذا طلب «نشر» المستودع:
   - **Publish repository**
   - اختر اسماً للمشروع (مثلاً `iif-fund-demo`)
   - **لا تضع** علامة على «Keep this code private» إن أردت موقعاً عاماً (أو اختر خاصاً إن أردت).
4. بعد النشر، كودك أصبح على **GitHub**.

> **ملاحظة:** إذا كان المجلد فيه `git` مسبقاً، قد يظهر لك التاريخ والـ commit — هذا طبيعي.

> **هذا المشروع جاهز على GitHub باسم:** [`alzahrani6020/iif-web`](https://github.com/alzahrani6020/iif-web) — إن كان المستودع مضافاً في GitHub Desktop، يكفي **Fetch** ثم **Push** عند التعديل.

### د) ربط Netlify بموقعك

1. سجّل في [netlify.com](https://www.netlify.com) (يمكن الدخول بحساب GitHub).
2. **Add new site** → **Import an existing project**.
3. اختر **GitHub** واسمح لـ Netlify بالوصول للمستودعات.
4. اختر المستودع **`alzahrani6020/iif-web`** (أو المستودع الذي أنشأته أنت).
5. **لا تغيّر** حقول البناء إن ظهرت تلقائياً — المشروع فيه ملف `netlify.toml` يوجّه Netlify.
6. اضغط **Deploy site** وانتظر دقيقة أو اثنتين.
7. سيعطيك Netlify **رابطاً** (مثل `random-name-123.netlify.app`) — هذا موقعك على الإنترنت.

**قائمة تحقق مفصّلة:** [NETLIFY-CHECKLIST.md](./NETLIFY-CHECKLIST.md)

### هـ) بديل بسيط: GitHub Pages (بدون Netlify)

إذا توقف Netlify أو أردت رابطاً على نطاق GitHub:

1. افتح المستودع على الويب (مثلاً [alzahrani6020/iif-web](https://github.com/alzahrani6020/iif-web)).
2. **Settings** → **Pages** (في الشريط الجانبي).
3. تحت **Build and deployment** اختر المصدر: **GitHub Actions** (وليس «Deploy from a branch» إن ظهر). إذا بقي الرابط `github.io/...` يعرض «لا يوجد موقع»، أنشئ الموقع لمرة واحدة عبر واجهة GitHub أو (للمستخدمين المتقدمين) بعد تثبيت **GitHub CLI** وتسجيل الدخول: `gh api -X POST repos/alzahrani6020/iif-web/pages -f build_type=workflow` أو السكربت `scripts\enable-github-pages.ps1`.
4. ادفع التغييرات إلى الفرع **main** (`Push`) — أو من تبويب **Actions** شغّل يدوياً سير العمل **Deploy GitHub Pages**.
5. بعد نجاح التشغيل (دقيقة تقريباً): الموقع على شكل `https://اسم-المستخدم.github.io/اسم-المستودع/`

التفاصيل التقنية في [README.md](./README.md) قسم «GitHub Pages».

## ماذا يحدث عندما تعدّل الملفات لاحقاً؟

1. عدّل الملفات على جهازك كالعادة.
2. في **GitHub Desktop**: ستظهر التغييرات → اكتب وصفاً قصيراً → **Commit to main** → **Push origin**.
3. **Netlify** يعيد بناء الموقع تلقائياً عند كل **Push** (إن كان مربوطاً بـ GitHub).
4. **GitHub Pages** (إن فعّلته كما في القسم «هـ») يعيد النشر تلقائياً عند تعديل ملفات واجهة الصندوق في المشروع.

## إذا واجهت مشكلة

| المشكلة | ماذا تجرب |
|---------|-----------|
| لا أفهم «أوامر الطرفية» | استخدم **GitHub Desktop** فقط في البداية. |
| Netlify لا يجد المستودع | تأكد أنك نشرت المشروع من Desktop وأنك منحت Netlify صلاحية GitHub. |
| البناء فشل (Build failed) | انسخ رسالة الخطأ من Netlify وأرسلها لمن يدعمك، أو راجع ملف `netlify.toml` في المشروع. |

---

## ملخص في سطرين

1. ارفع المجلد إلى **GitHub** (أسهل مع **GitHub Desktop**).  
2. في **Netlify** اربط نفس المستودع واضغط Deploy — يظهر لك رابط الموقع.

لا تحتاج أن تحفظ كل المصطلحات؛ المهم: **GitHub = مكان الكود، Netlify = الموقع الذي يفتحه الزوار.**

# للبدء — ما الذي **عليك أنت** فعله (مرة واحدة أو نادراً)

> باقي المشروع (كود، Netlify، GitHub، قوائم التحقق) **موجود في المستودع**.  
> هذا الملف يخص **جهازك وحساباتك فقط** — لا يستطيع أحد آخر تنفيذها عنك.

---

## 1) على جهازك (ضروري للتطوير المحلي)

| الخطوة | لماذا | أين |
|--------|--------|-----|
| **تثبيت Node.js 20 أو أحدث** | تشغيل `npm` والبناء | [nodejs.org](https://nodejs.org) — نسخة LTS |
| **تثبيت Git** | رفع الكود والمزامنة مع GitHub | [git-scm.com/download/win](https://git-scm.com/download/win) |
| **فتح المشروع في Cursor** | المساعد يقرأ الملفات من **المجلد المفتوح** | File → Open Folder → مجلد `iif-fund-demo` |

بعدها من الطرفية داخل المجلد:

```bash
npm ci
npm run build
npm start
```

إذا نجحت الأوامر — البيئة **جاهزة للتطوير**.

---

## 2) حساب GitHub (ضروري إن أردت نسخة على السحابة)

| الخطوة | لماذا |
|--------|--------|
| إنشاء حساب على **GitHub** | حفظ الكود ومزامنة الأجهزة |
| **Clone** أو **Push** للمستودع `alzahrani6020/iif-web` | حسب ما إذا نسخت المشروع من عندك أم من GitHub |

أسهل للمبتدئين: **GitHub Desktop** — تسجيل دخول، ثم Clone أو Publish.

---

## 3) الموقع على الإنترنت (Netlify) — اختياري لكنها «البدء» للزوار

| الخطوة | لماذا |
|--------|--------|
| حساب على **Netlify** (غالباً بالدخول بـ GitHub) | استضافة الموقع الثابت |
| **Import from Git** → اختيار مستودع `iif-web` | النشر من الكود تلقائياً عند كل `push` |

الإعداد الفني في **`netlify.toml`** — لا تحتاج تكرارها يدوياً إن قرأت Netlify الملف.

> **أسرار GitHub Actions** (`NETLIFY_AUTH_TOKEN` و `NETLIFY_SITE_ID`) **للنشر اليدوي من Actions فقط** — إن لم تستخدم workflow «Deploy Netlify»، **لا تضطر** لإضافتها.

---

## 4) تحسين تجربة Cursor (اختياري)

| الخطوة | لماذا |
|--------|--------|
| نسخ **`cursorignore.example`** إلى **`.cursorignore`** في جذر المشروع | فهرسة أسرع وأقل ضوضاء |

---

## 5) ملخص: «هل أنا جاهز؟»

- **تطوير محلي:** Node + Git + `npm ci` + `npm start` تعمل → **نعم**.
- **نسخة على GitHub:** حساب + push/clone → **نعم**.
- **موقع للعامة:** Netlify مربوط بالمستودع + deploy ناجح → **نعم**.

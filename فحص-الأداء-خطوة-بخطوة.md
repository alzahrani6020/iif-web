# فحص الأداء والوظائف — خطوة بخطوة (معاً)

استخدم هذا الدليل للمرور على الموقع **واحدة تلو الأخرى** وتسجيل النتيجة (✓ / ✗).

---

## تجهيز (مرة واحدة لكل جلسة)

| # | الخطوة | ماذا تتوقع |
|---|--------|------------|
| 1 | من جذر المشروع: `npm run verify` | ينتهي بدون خطأ (بناء بيانات الحكومة + فحص السكربتات). |
| 2 | تشغيل الخادم: `npm start` | يظهر في الطرفية: المنفذ **3333** وبروكسي SearXNG إن وُجد. |
| 3 | في طرفية ثانية: `npm run check-urls` | كل المسارات **OK 200** (أو اختياري فقط لـ SearXNG إن لم يُشغَّل Docker). |

> **تلقائياً:** بعد `npm start` يمكن تشغيل `npm run check-urls` للتحقق السريع من أن الصفحات تُجاب.

| A2 | […/iif-fund-demo/index.html](http://127.0.0.1:3333/financial-consulting/iif-fund-demo/index.html) | نفس الواجهة بمسار مباشر؛ الشعار والخطوط سليمة. |
| A3 | […/fund-site/index.html](http://127.0.0.1:3333/financial-consulting/fund-site/index.html) | صفحة مبسّطة + iframe للمنصة الحكومية إن وُجد. |

## ب) المنصة الحكومية — سلوك الواجهة

| # | الإجراء | ماذا تتوقع |
|---|---------|------------|
| B1 | «عرض الكل» | قائمة نتائج محلية (عدد كبير من الجهات). |
| B2 | كتابة كلمة في البحث ثم «بحث» | نتائج تتغير؛ لا تعطل الصفحة. |
| B3 | فلتر الدولة / النوع | يضيّق النتائج. |
| B4 | بطاقة جهة — «الموقع الإلكتروني» | يفتح رابطاً في تبويب جديد (إن وُجد رابط). |
| B5 | «نسخ رابط البحث» | يظهر إشعار نجاح أو لا يتعطل. |
| B6 | تضمين ويب (مربّع SearXNG) — إن مفعّلاً | نتائج ويب أسفل المحلية؛ إن فشل يظهر رسالة توضيحية وليس كسراً كاملاً. |

## ج) الأداء السريع (اختياري)

| # | الأداة | ملاحظة |

## د) محرك البحث (SearXNG) — اختياري

| # | الخطوة | ماذا تتوقع |
|---|--------|------------|
| D1 | `cd engines/searxng` ثم `docker compose up -d` | واجهة المحرك [127.0.0.1:8080](http://127.0.0.1:8080) تعمل. |
| D2 | `npm run check-urls` | سطر `/api/searx/` **OK 200** إن المحرك يعمل. |
| D3 | من المنصة الحكومية: بحث + تضمين ويب | نتائج JSON من البروكسي. |

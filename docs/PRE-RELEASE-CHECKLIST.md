# قائمة ما قبل الإصدار (سريعة — ~10 دقائق)

استخدمها بعد دمج كبير على `main` أو قبل إرسال رابط رسمي لجهة خارجية.

## 1) البناء والمستودع

- [ ] **CI أخضر** على آخر commit في [Actions](https://github.com/alzahrani6020/iif-web/actions) (سير عمل **CI** / مهمة **verify**).
- [ ] محلياً (اختياري): `npm run verify` من جذر المشروع.

## 2) GitHub Pages

- [ ] افتح [الموقع على Pages](https://alzahrani6020.github.io/iif-web/) — تحميل الصفحة الرئيسية بدون أخطاء واضحة.
- [ ] **Ctrl+F5** أو نافذة خاصة إن بدا الكاش قديماً.
- [ ] تذكّر: **`/healthz`** و**`/api/*`** غير متوفرة هنا (موقع ثابت فقط).

## 3) Netlify (إن كان الموقع النشط هناك)

- [ ] افتح موقع Netlify المعتمد (راجع [CANONICAL-URLS.md](../CANONICAL-URLS.md)).
- [ ] جرّب مسار **`/fund`** أو المعادل عندكم.
- [ ] راجع [NETLIFY-FREE-TIER.md](../NETLIFY-FREE-TIER.md) إن اقتربتم من حدود البناء.

## 4) اللوحة والدخول

- [ ] تسجيل دخول اختباري (حساب مسموح).
- [ ] فتح اللوحة عبر **`#dashboard`** أو معاملات الاستعلام المعتمدة في [CANONICAL-URLS.md](../CANONICAL-URLS.md).

## 5) مراقبة التوفر (تلقائي + اختياري)

- [ ] **مفعّل في المستودع:** سير العمل **Uptime ping** (كل ~6 ساعات + تشغيل يدوي من Actions). عند سقوط الرابط يفشل التشغيل — فعّل إشعارات البريد لـ **Actions** من إعدادات حساب GitHub إن رغبت.
- [ ] (اختياري) خدمة خارجية مثل **UptimeRobot** أو **Better Stack** لنفس الروابط إن أردت لوحة وSMS منفصلة.

## 6) حجم الواجهة (صيانة دورية)

- [ ] `npm run report:index-size` — إن كان الملف ضخماً جداً، خطط لتقسيم تدريجي (مكوّنات / تحميل كسول).

---

**الرابط الرسمي للجهات:** وحّد المصدر في [CANONICAL-URLS.md](../CANONICAL-URLS.md) ولا ترسل أكثر من نطاق «إنتاج» واحد دون توضيح.
